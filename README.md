# India Mushroom Days 2027 — Event Website (separated build)

Same site as before, now split into separate HTML, CSS, and JS files, with a
**dynamic header and footer** that render from a single config object.

## Structure

```
index.html            Page markup. Links the CSS + JS. The header and footer
                      are empty placeholders that get filled in by JavaScript.
css/
  styles.css          All styles.
js/
  frames.js           List of the 40 hero animation frame paths (window.FRAMES).
  components.js        Dynamic header + footer. Edit SITE_CONFIG here.
  app.js              All page logic: scroll animation, countdown, schedule
                      filters, speaker search, FAQ, reveals, etc.
frames/
  frame001.jpg … frame040.jpg   The hero animation frames.
```

## Dynamic header & footer

Both the navigation bar and the footer are generated by `js/components.js`
from one object called `SITE_CONFIG` at the top of that file. Change a nav link,
the contact email, a footer column, or the social links in `SITE_CONFIG` and
**both the header and footer update automatically** — you never edit the markup
in two places.

The placeholders they mount into live in `index.html`:

```html
<header id="site-header"></header>
...
<footer id="site-footer"></footer>
```

To add a nav item, for example, add one line to `SITE_CONFIG.nav`:

```js
{ label: "Sponsors", href: "#sponsors" }
```

## Running it

Because the page loads files over relative paths, open it through a local web
server rather than double-clicking the file:

```bash
cd this-folder
python3 -m http.server 8000
# then visit http://localhost:8000
```

To publish, upload the whole folder to any static host (Netlify, GitHub Pages,
S3, etc.). Keep the folder structure intact so the `css/`, `js/`, and `frames/`
paths resolve.

## Notes

- The only external dependency is Google Fonts (Space Grotesk, Inter, Instrument
  Serif), loaded over the network.
- Copy (names, organizations, prices, contact details) is placeholder content —
  replace with your real event details before going live.
- Accessibility: keyboard focus, ARIA labels, accessible mobile menu, and
  reduced-motion support are built in.
